# 产品需求

## 产品定位

本产品不是通用 AI 写作工具，而是“向上汇报质量提升工具”。它帮助用户把真实工作材料转成领导可读、可判断、可追踪的日报、周报和月报。

## 用户画像

- 一线执行者：每天做了很多事，但不知道怎么表达价值。
- 跨团队协作者：需要把阻塞、依赖和风险讲清楚。
- 销售/业务人员：需要围绕商机、金额、客户反馈和回款风险汇报。
- 对 AI 有兴趣但担心隐私的个人用户：希望数据先留在本机。

## 核心场景

1. 日报：当天工作材料整理成可直接发给直属领导的文本。
2. 周报：从历史日报和补充重点中归纳本周成果、风险和下周计划。
3. 月报：从历史周报/日报中总结阶段成果、趋势和下月重点。
4. 优化：根据评分结果修复低分项，尤其是成果表达、数据支撑和风险透明度。
5. 粘贴拆解：用户可先粘贴会议纪要、流水记录、客户沟通或缺陷记录，再自动拆成当前岗位的结构化字段草稿。
6. 格式交付：同一份报告支持 Markdown、IM 简洁版、邮件版三种复制格式。

## 岗位预设

- 产品经理：指标变化、需求推进、上线结果、用户反馈、决策事项、跨团队依赖。
- 前端：页面/组件交付、交互体验、性能优化、兼容性、线上问题、联调阻塞。
- 后端：接口/服务交付、稳定性、性能、数据、架构改造、故障风险。
- 测试：测试范围、用例执行、缺陷分布、阻塞问题、质量风险、上线建议。
- 业务销售：线索、拜访、商机阶段、成交金额、回款风险、客户反馈、下周推进。

## 老板人设

老板人设由标签+权重组成：

- 结果导向
- 数据敏感
- 风险敏感
- 喜欢简洁
- 关注过程
- 关注客户
- 关注成本
- 关注协作

生成和评分时必须根据权重改变报告重点，例如风险敏感型老板要更早看到风险、影响范围和预案。

## 评分标准

每份报告必须评分：

- 老板适配度
- 成果表达
- 数据支撑
- 结构清晰度
- 风险透明度
- 下一步可执行性
- 事实忠实度

不同岗位使用不同权重，销售更重收入与推进，测试更重风险与覆盖，后端更重稳定性与影响面。

## 反虚假原则

- 不编造数据、客户、收入、上线结果或收益。
- 没有量化数据时写“暂无量化数据”或提示补充。
- 优化稿只能基于原始输入改写，不扩大成果。
- 本地体验稿必须明确是基于已填写事实整理。

## v1 上线范围

包含：

- 本地 Web App
- 多岗位输入表单
- 老板人设编辑
- 多 AI provider 配置
- 报告生成、评分、优化
- SQLite 历史沉淀
- 周报/月报历史聚合
- 数据导出、清空、健康检查
- 大段材料拆解成岗位字段草稿
- Markdown、IM、邮件三种复制格式

不包含：

- 团队账号与权限
- 云端同步
- 飞书/钉钉/Jira/Git/CRM 自动同步
- Word/PDF 导出
- 自动定时提醒

## 成功指标

- 用户第一次打开后 5 分钟内能生成一份可复制报告。
- 至少 5 个岗位预设字段明显不同。
- 无 API Key 时能完成体验，有 API Key 时能完成真实 AI 生成。
- 周报/月报能引用历史报告，而不是只靠手动输入。
- 用户能导出和清空本地数据。
